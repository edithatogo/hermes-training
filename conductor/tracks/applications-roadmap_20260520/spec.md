# Specification: Applications Roadmap and Model-Use Expansion

## Overview

Identify additional uses for the trained and runtime-only models so the project can produce practical local AI components beyond a single chat adapter.

## Requirements

- Map model families to applications.
- Distinguish chat SFT, retrieval, tool-call validation, and teacher/evaluator workflows.
- Identify improvements that should become future tracks.
- Keep claims tied to benchmark gates.

## Acceptance Criteria

- `APPLICATIONS.md` exists and is linked from README/HANDOFF/CONDUCTOR.
- At least three future applications are eligible to become Conductor tracks.
- Publication guidance distinguishes GitHub docs from Hugging Face model/dataset artifacts.

## Out of Scope

- Implementing every downstream application immediately.
- Building a UI.
